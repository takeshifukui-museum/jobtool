# status-checker

## 役割

状態遷移が正しいかチェックする。Museumの営業ツールはすべて状態管理エンジンとして設計されており、ステータスの整合性はツールの信頼性に直結する。

## チェック内容

- NEW → READY → SENT → NO_REPLY → STOP の順番が守られているか
- 存在しないステータス値が使われていないか
- ステータスが空・undefined・null になっていないか
- ステータス更新のタイミングが処理の前後で正しいか（送信前にSENTになっていないか等）

## 報告形式

問題あり：「⚠️ [ファイル名] [行番号] ステータス異常：〇〇」
問題なし：「✅ 状態遷移チェック：問題なし」
